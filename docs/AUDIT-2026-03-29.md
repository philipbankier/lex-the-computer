## Phase 0: Scaffolding & Infrastructure

### Frontend Framework: Next.js 15
- **Status**: ⚠️ OUTDATED — **Next.js 16.2** is current (released Oct 2025, latest 16.2 March 2026)
- Next.js 16 brought: Turbopack as default bundler, new caching model, React 19.2, Devtools upgrades, Build Adapters API
- Next.js 16.2 adds: Adapters (easier self-hosting beyond Vercel), rendering perf improvements, better error pages
- **Verdict**: Should upgrade to Next.js 16. The Adapters API is important for self-hosted deployment which is Lex's whole model.
- **Action**: `npm install next@latest` — likely needs caching behavior updates since Next.js 16 changed the caching model

### React: React 19
- **Status**: ⚠️ **React 19.2** is current (ships with Next.js 16)
- React 19.2 includes refined Server Components, improved streaming
- **Action**: Upgrade alongside Next.js 16

### Backend: Hono (Node.js)
- **Status**: ✅ CORRECT CHOICE — Hono is THE TypeScript backend framework in 2026
- ~1.8M weekly npm downloads, 4x faster than Express
- TypeScript-first, runs on Node.js, Bun, Deno, Cloudflare Workers, AWS Lambda
- The "Stop Using Express.js in 2026" consensus is real
- **Action**: No change needed. Good pick.

### Database: PostgreSQL + Drizzle ORM
- **Status**: ✅ CORRECT CHOICE — Drizzle is the 2026 consensus best ORM
- "Drizzle is the best default for new projects" — multiple 2026 sources
- 3-5x faster than Prisma, 70% smaller bundle, SQL-level control with full type safety
- **Action**: No change needed. Excellent pick.

### Cache/Queue: Redis + BullMQ
- **Status**: ✅ CORRECT CHOICES
- BullMQ is "the de-facto standard job queue for Node.js in 2026"
- BullMQ 5.71 is current with OpenTelemetry support
- Redis Streams backing is solid
- **Action**: Ensure we're on BullMQ 5.x. No architecture change needed.

### Auth: Better Auth
- **Status**: ✅ CORRECT CHOICE — "For new projects in 2026, Better Auth offers the best balance"
- TypeScript-first, framework-agnostic, plugin system, 2FA built-in
- Growing faster than NextAuth/Clerk alternatives for self-hosted use cases
- **Action**: No change needed.

### AI Proxy: LiteLLM
- **Status**: 🟡 STILL VALID but alternatives emerging
- LiteLLM remains the standard for multi-provider routing
- Alternatives: Bifrost (Go, faster), Portkey (production observability), OpenRouter
- For our use case (self-hosted, Docker Compose), LiteLLM is still the best fit
- **BUT**: Zo's approach is fundamentally different — they support Claude Code, Codex, and Gemini CLI as first-class providers (not just API key routing). See Phase 1 notes.
- **Action**: Keep LiteLLM for API key routing. ADD Claude Code/Codex/Gemini CLI as separate provider types.

### Monorepo: pnpm + Turborepo
- **Status**: ✅ CORRECT CHOICE
- "For most new monorepos in 2026, pnpm workspaces is the pragmatic choice"
- Turborepo: ~2M weekly downloads, fast build caching, minimal config
- **Action**: No change needed.

### Reverse Proxy: Caddy
- **Status**: ✅ CORRECT CHOICE for our use case
- "Caddy wins on simplicity" — auto HTTPS, minimal config, perfect for self-hosted
- Traefik is better for dynamic container discovery (more relevant for multi-user mode)
- **Action**: Keep Caddy for single-user. Consider Traefik for Phase 11 multi-user.

### UI: Tailwind CSS 4 + shadcn/ui
- **Status**: ✅ CORRECT CHOICES
- Tailwind CSS 4: CSS-native engine, dropped tailwind.config.js for @theme directives, faster builds
- shadcn/ui: "go-to React UI library" with Radix primitives, dominant for SaaS dashboards
- **Action**: Ensure we're actually on Tailwind CSS 4 (not 3). V4 has breaking changes.

### Docker Compose
- **Status**: ✅ Appropriate for self-hosted single-user deployment
- **Action**: No change needed.

## Phase 1: Chat

### Vercel AI SDK
- **Status**: ⚠️ NEEDS UPDATE — **Vercel AI SDK v6** is current
- V6 introduces: Server Actions (replacing REST), refined hooks (useChat, useCompletion), robust server-side tool calling, AI Gateway with hundreds of models
- Our build likely used AI SDK v4 or v5 patterns
- **Action**: Upgrade to AI SDK v6. Update streaming to use Server Actions pattern.

### AI Chat UI
- **Status**: 🟡 REVIEW NEEDED
- 2026 ecosystem includes assistant-ui (Radix headless pattern with Thread, Composer, Message primitives), which provides streaming-first chat components
- **Action**: Evaluate if our custom chat UI matches quality of assistant-ui primitives

### AI Provider Architecture
- **Status**: 🔴 FUNDAMENTALLY INCOMPLETE
- We only have: LiteLLM + BYOK API keys
- **Zo has**: Built-in credits, BYOK API keys, Claude Code subscription, Codex subscription, Gemini CLI subscription
- Users can run `claude`, `codex`, or `gemini` in terminal, authenticate, then select as AI provider
- This is a MAJOR differentiator — "Settings → AI Providers" not just "BYOK keys"
- **Action**: Add Claude Code, Codex, Gemini CLI as first-class AI provider types. Model picker shows provider source.

### Missing Feature: "Edit file fast"
- Zo has `edit-file` (precise edits) AND `edit-file-llm` (fast apply using LLM)
- We likely only have precise edits
- **Action**: Add LLM-powered fast file editing tool

## Phase 10: Advanced Features

### Image Generation
- **Status**: 🔴 OUTDATED
- We built: DALL-E 3, Stability AI
- 2026 leaders: FLUX.2 [pro], GPT Image 1.5, Nano Banana 2 (Google), Seedream V4.5, Recraft V3, Ideogram V3

- fal.ai SDK: `@fal-ai/client`, 6 lines of code to generate
- **Action**: Replace with fal.ai. Support model selection (FLUX.2 default, GPT Image, Recraft for text/vector)

### Image Editing
- **Status**: 🔴 OUTDATED
- Zo has `edit-image` tool for remixing/editing images
- fal.ai has editing models (Seedream editing, Recraft editing)
- **Action**: Replace with fal.ai editing models

### Video Generation
- **Status**: 🔴 OUTDATED
- Sora is SHUTTING DOWN
- 2026 leaders: Google Veo 3.1, Kling AI 3.0, Runway Gen-4.5, Seedance 2.0
- fal.ai has: Veo 3.1, Kling 3.0, and others
- **Action**: Replace with fal.ai video models

### Audio Transcription
- **Status**: 🟡 PARTIALLY OUTDATED
- OpenAI Whisper API still works but is slow
- **Groq Whisper**: 228x real-time speed, $0.111/hour — same model on Groq's LPU chips
- NVIDIA Parakeet V3 also excellent
- **Action**: Add Groq Whisper as primary option (env: `GROQ_API_KEY`). Keep OpenAI Whisper as fallback.

### Video Transcription
- **Status**: 🟡 Same as audio — ffmpeg extract + transcription
- **Action**: Use Groq Whisper for the transcription step

### MCP Server
- **Status**: 🔴 LIKELY WRONG IMPLEMENTATION
- Current MCP spec: **2025-11-25** (stable)
- Official TypeScript SDK: `@modelcontextprotocol/sdk` (v1.x)
- Transport: **Streamable HTTP** (not SSE, not custom)
- Zo's endpoint: `https://api.zo.computer/mcp` with Bearer token auth
- SDK has `McpServer` class and `StreamableHTTPServerTransport`
- Config examples needed for: Claude Code, Claude Desktop, Cursor, Zed, OpenCode, Gemini CLI
- **Action**: Rewrite using `@modelcontextprotocol/sdk` with `StreamableHTTPServerTransport`. Match Zo's `/mcp` endpoint pattern exactly.

### Browser Automation
- **Status**: 🟡 COULD BE BETTER
- We built: Raw Playwright
- 2026 leaders for AI browser automation:
  - **Browser Use** — 81k GitHub stars, 89.1% benchmark, Python-first
  - **Stagehand** (Browserbase) — TypeScript, Playwright-based, `act()/extract()/observe()` natural language
  - **Playwright MCP** — Microsoft's official MCP server, uses accessibility tree
- For a TypeScript project: **Stagehand** is ideal — drop-in Playwright enhancement
- **Action**: Add Stagehand on top of Playwright for AI-powered interactions. Consider exposing browser via Playwright MCP as well.

### SSH: ssh2
- **Status**: ✅ ssh2 is still the standard Node.js SSH library
- Zo has SSH support (`ssh-computer.md`, `ssh-zo.md`)
- **Action**: No change needed

### Diagrams: D2
- **Status**: ✅ D2 is still appropriate
- Zo uses D2 (`generate-d2-diagram` tool)
- **Action**: No change needed

### Google Maps
- **Status**: ✅ Zo has `maps-search` tool
- **Action**: No change needed

### Missing Tools from Zo
- 🔴 `find-similar-links` — find web pages similar to a given URL
- 🔴 `image-search` — search for images on the web
- 🔴 `change-hardware` — change GPU/CPU/RAM configuration
- 🔴 `set-active-persona` — set persona per channel
- 🟡 `update-user-settings` — update name/bio settings via tool
- **Action**: Add these missing tools

### Missing Feature: AI Providers (Claude Code / Codex / Gemini CLI)
- 🔴 **THIS IS THE BIGGEST GAP**
- Zo lets users:
  1. Open terminal, run `claude` / `codex` / `gemini`
  2. Authenticate with their subscription
  3. Go to Settings → AI Providers, enable the provider
  4. Select it from model picker in chat
- This means users with a $20/mo Claude Pro plan get full agentic coding for free
- **Action**: Implement AI Providers architecture:
  - Settings → AI Providers page (not just "BYOK Keys")
  - Detect installed CLIs (claude, codex, gemini)
  - Auth flow via terminal
  - Model picker shows: Built-in | Claude Code | Codex | Gemini CLI | BYOK
